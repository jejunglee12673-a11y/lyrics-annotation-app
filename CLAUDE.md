# lyrics-annotation-app

歌詞にアノテーション（注釈・コメント）を付けるWebアプリケーション。

## Git 運用ルール

**コードを変更するたびに必ず GitHub へプッシュすること。**

```bash
git add <変更ファイル>
git commit -m "変更内容の説明"
git push origin main
```

- コミットメッセージは変更の「なぜ」を簡潔に書く（何をしたかはdiffで分かる）
- `git add -A` や `git add .` は避け、変更ファイルを個別に指定する
- `.env` や認証情報を含むファイルは絶対にコミットしない
- force push は禁止（`git push --force` は使わない）
- main ブランチへ直接プッシュしてよいのは軽微な修正のみ
- 機能追加・大きな変更はブランチを切って PR を作成する

## 開発方針

- 機能追加・バグ修正・リファクタは 1 コミットにまとめず、目的ごとに分けてコミットする
- テストが通っている状態でのみプッシュする
- コードレビューなしで main にマージする場合も、ローカルで動作確認を必ず行う

## コーディング規則

- コメントは「なぜそうしているか」が自明でない箇所にのみ書く
- 使われていないコードは削除する（コメントアウトして残さない）
- セキュリティ: ユーザー入力は必ずバリデーション・サニタイズを行う
- 環境変数は `.env` に定義し、`.env.example` にキー名だけ記載する
